## 📋 全局约定

### 统一返回格式
```json
{
  "code": 200,
  "message": "Success",
  "data": { ... }
}
```

错误码说明

状态码 说明
200 成功
400 请求参数错误
401 未授权 / Token无效
403 权限不足
404 资源不存在
500 服务器错误

认证方式

除登录接口外，所有请求都需要在 Header 中添加：

```
Authorization: Bearer <token>
```

---

一、认证接口

1.1 用户登录

```http
POST /api/auth/login
Content-Type: application/json
```

请求参数

```json
{
  "username": "teacher_zhang",
  "password": "123456",
  "type": "teacher"   // teacher / admin / parent
}
```

成功响应示例

```json
{
  "code": 200,
  "message": "Login successful",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "user": {
      "id": "teacher_004",
      "name": "张老师",
      "role": "teacher"
    }
  }
}
```

测试用例

```bash
# 教师登录
curl -X POST "http://122.51.12.83:8000/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"username":"teacher_zhang","password":"123456","type":"teacher"}'

# 管理员登录
curl -X POST "http://122.51.12.83:8000/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123","type":"admin"}'

# 家长登录
curl -X POST "http://122.51.12.83:8000/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"username":"zhangsan","password":"123456","type":"parent"}'
```

二、家长端接口

2.1 获取任务列表

```http
GET /api/parent/tasks
Authorization: Bearer <token>
```

成功响应示例

```json
{
  "code": 200,
  "data": {
    "pending": [...],
    "completed": [...]
  }
}
```

测试用例

```bash
# 先获取家长token
PARENT_TOKEN=$(curl -s -X POST "http://122.51.12.83:8000/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"username":"zhangsan","password":"123456","type":"parent"}' | grep -o '"token":"[^"]*"' | cut -d'"' -f4)

# 获取任务列表
curl -X GET "http://122.51.12.83:8000/api/parent/tasks" \
  -H "Authorization: Bearer $PARENT_TOKEN"
```
