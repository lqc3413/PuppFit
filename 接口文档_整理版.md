## 2. 全局约定

### 2.1 统一返回格式

```json
{
  "code": 200,
  "message": "Success",
  "data": {}
}
```

### 2.2 错误码说明

| 状态码 | 说明 |
| --- | --- |
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权 / Token 无效 |
| 403 | 权限不足 |
| 404 | 资源不存在 |
| 500 | 服务器错误 |

### 2.3 认证方式

除登录接口外，所有请求都需要在请求头中携带：

```http
Authorization: Bearer <token>
```

### 2.4 测试说明

- 下文中的 `curl` 示例沿用原文风格，默认使用命令行变量保存登录返回的 `token`。
- `PARENT_TOKEN`、`TEACHER_TOKEN`、`ADMIN_TOKEN` 分别代表家长、教师、管理员登录后的访问令牌。

## 3. 认证接口

### 3.1 用户登录

**用途**  
获取当前用户身份对应的登录令牌与基础用户信息。

**请求**

```http
POST /api/auth/login
Content-Type: application/json
```

**请求体**

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `username` | string | 是 | 登录用户名 |
| `password` | string | 是 | 登录密码 |
| `type` | string | 是 | 用户类型，可选 `teacher`、`admin`、`parent` |

**请求示例**

```json
{
  "username": "teacher_zhang",
  "password": "123456",
  "type": "teacher"
}
```

**成功响应示例**

```json
{
  "code": 200,
  "message": "Login successful",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "user": {
      "id": "teacher_004",
      "name": "张老师",
      "role": "teacher"
    }
  }
}
```

**测试示例**

```bash
# 教师登录
curl -X POST "http://122.51.12.83:8000/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"username":"teacher_zhang","password":"123456","type":"teacher"}'

# 管理员登录
curl -X POST "http://122.51.12.83:8000/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123","type":"admin"}'

# 家长登录
curl -X POST "http://122.51.12.83:8000/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"username":"zhangsan","password":"123456","type":"parent"}'
```

## 4. 家长端接口

### 4.1 获取任务列表

**用途**  
获取当前家长账号对应的任务列表，按待完成和已完成分类返回。

**请求**

```http
GET /api/parent/tasks
Authorization: Bearer <token>
```

**请求参数**  
无

**成功响应示例**

```json
{
  "code": 200,
  "data": {
    "pending": [],
    "completed": []
  }
}
```

**测试示例**

```bash
# 先获取家长 token
PARENT_TOKEN=$(curl -s -X POST "http://122.51.12.83:8000/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"username":"zhangsan","password":"123456","type":"parent"}' | grep -o '"token":"[^"]*"' | cut -d'"' -f4)

# 获取任务列表
curl -X GET "http://122.51.12.83:8000/api/parent/tasks" \
  -H "Authorization: Bearer $PARENT_TOKEN"
```

### 5.2 发布任务

**用途**  
教师向指定学生设备发布运动任务。

**请求**

```http
POST /api/teacher/publish_task
Content-Type: application/json
Authorization: Bearer <token>
```

**请求体**

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `device_id` | string | 是 | 设备 ID |
| `user_id` | string | 是 | 学生用户 ID |
| `target_count` | number | 是 | 目标次数 |
| `duration_limit` | number | 是 | 时长限制，单位未在原文注明 |

**请求示例**

```json
{
  "device_id": "puppy_001",
  "user_id": "123456",
  "target_count": 30,
  "duration_limit": 60
}
```

**成功响应示例**

```json
{
  "code": 200,
  "message": "发布成功",
  "data": {
    "id": 1772704250877,
    "title": "新任务",
    "deadline": "2026-03-05"
  }
}
```

**测试示例**

```bash
curl -X POST "http://122.51.12.83:8000/api/teacher/publish_task" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TEACHER_TOKEN" \
  -d '{"device_id":"puppy_001","user_id":"123456","target_count":30,"duration_limit":60}'
```
