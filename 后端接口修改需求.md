# 后端接口修改需求

> **日期**：2026-03-06  
> **目标**：实现「教师发布设备任务 → 家长端看到待办 → 学生完成 → 教师端看到成绩」的完整链路

---

## 需要修改的接口（1 个）

### `GET /api/parent/tasks`

**当前行为**：`pending` 列表只返回预置的"教学任务"（如 id=1001 第一周体能测试），不包含教师通过 `publish_task` 发布的设备任务。

**期望行为**：`pending` 列表中应**包含该学生的设备任务**（即 `publish_task` 发布的、`user_id` 匹配当前学生、`status=created` 的任务）。

**期望返回格式**：

```json
{
  "code": 200,
  "data": {
    "pending": [
      {
        "id": 1772738262699,
        "title": "仰卧起坐训练",
        "deadline": "2026-03-06",
        "createdAt": "2026-03-06",
        "exercises": [
          {
            "type": "仰卧起坐",
            "target": 15,
            "unit": "个",
            "weight": 1.0
          }
        ]
      }
    ],
    "completed": [ /* 保持不变 */ ]
  }
}
```

**字段映射建议**（设备任务 → pending 格式）：

| pending 字段 | 来源 | 说明 |
|---|---|---|
| `id` | 设备任务的 `task_id` 或数字 id | 任务唯一标识 |
| `title` | 根据 `exercise_type` 生成 | 如 `"仰卧起坐训练"` |
| `deadline` | 设备任务的 `created_at` 或自定义 | 截止日期 |
| `createdAt` | 设备任务的 `created_at` | 创建时间 |
| `exercises` | 从设备任务构造 | `type`=运动名称, `target`=target_count, `unit`=对应单位, `weight`=1.0 |

**查询逻辑**：从设备任务表中查 `user_id = 当前学生` 且 `status = 'created'` 的记录，转换格式后合并到 `pending` 列表。
